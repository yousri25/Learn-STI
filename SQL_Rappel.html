Apprendre SQL

Un guide pas à pas pour créer, gérer et interroger une base de données

Chapitre 1 · Introduction

C'est quoi SQL ?

Avant de stocker des données dans une base, il faut comprendre ce qu'est SQL et comment les données sont organisées.

// Définition

SQL signifie Structured Query Language (langage de requêtes structurées). C'est le langage standard pour créer, modifier et interroger une base de données relationnelle comme MySQL ou MariaDB.

💡 Analogie

Une base de données est comme un classeur Excel. Chaque table est une feuille. Chaque colonne est un champ (Nom, Âge, Note…). Chaque ligne est un enregistrement (les données d'un élève). SQL est le langage qui permet de lire, ajouter ou modifier ces données.

Les 3 rôles de SQL

Les contraintes d'intégrité
Les contraintes s'ajoutent lors de la création d'une colonne pour garantir la qualité des données : empêcher les valeurs nulles, imposer l'unicité, définir des clés…
NOT NULL
La colonne ne peut pas contenir de valeur nulle. Obligatoire pour les champs essentiels.
nom VARCHAR(50) NOT NULL
DEFAULT valeur
Attribue une valeur par défaut si aucune valeur n'est fournie lors de l'insertion.
actif BOOLEAN DEFAULT true
CHECK (condition)
Vérifie que les valeurs respectent une condition logique avant insertion.
note DECIMAL CHECK (note BETWEEN 0 AND 20)
UNIQUE
Garantit que toutes les valeurs de la colonne sont différentes (pas de doublons).
email VARCHAR(100) UNIQUE
PRIMARY KEY PK
Identifiant unique et non nul de chaque ligne. Chaque table doit en avoir une.
id INT PRIMARY KEY AUTO_INCREMENT
FOREIGN KEY FK
Lien vers la clé primaire d'une autre table. Garantit la cohérence entre tables.
FOREIGN KEY (id_classe) REFERENCES classe(id_classe)
ON UPDATE CASCADE
Met à jour automatiquement la clé étrangère si la clé primaire référencée est modifiée.
ON DELETE CASCADE
Supprime automatiquement les enregistrements de la table fille si l'enregistrement parent est supprimé.

Chapitre 3 · DDL — Définition des données

Créer & Modifier la structure

Le DDL (Data Definition Language) permet de créer des bases, des tables et de modifier leur structure avec CREATE, ALTER et DROP.

CREATE — Créer une base et une table
DDL_create.sql
-- Créer une base de données
CREATE DATABASE ecole;

-- Créer une table avec types et contraintes
CREATE TABLE eleve (
  id_eleve  INT           PRIMARY KEY AUTO_INCREMENT,
  nom       VARCHAR(50)  NOT NULL,
  prenom    VARCHAR(50)  NOT NULL,
  email     VARCHAR(100) UNIQUE,
  note      DECIMAL(4,2) CHECK (note BETWEEN 0 AND 20),
  id_classe INT,
  CONSTRAINT fk_classe
    FOREIGN KEY (id_classe) REFERENCES classe(id_classe)
    ON UPDATE CASCADE ON DELETE CASCADE
);
ALTER — Modifier une table existante
CommandeAction
ALTER TABLE t ADD col type [contrainte]Ajouter une nouvelle colonne
ALTER TABLE t DROP colSupprimer une colonne
ALTER TABLE t ALTER* col typeModifier le type d'une colonne (* ou MODIFY)
ALTER TABLE t RENAME** nom1 TO nom2Renommer une colonne (** ou CHANGE)
ALTER TABLE t ADD CONSTRAINT nom cAjouter une contrainte sur une colonne
ALTER TABLE t DROP CONSTRAINT nomSupprimer une contrainte
ALTER TABLE t ENABLE CONSTRAINT nomActiver une contrainte

DROP — Supprimer
DDL_drop.sql
-- Supprimer une table
DROP TABLE eleve;

-- Supprimer une base de données
DROP DATABASE ecole;

-- Exemple ALTER TABLE complet
ALTER TABLE eleve
  ADD telephone VARCHAR(15);

ALTER TABLE eleve
  DROP telephone;
Chapitre 4 · Lire les données

La requête SELECT

SELECT est la commande la plus importante — elle permet de lire et filtrer les données d'une ou plusieurs tables.

Structure complète d'un SELECT
SELECT_structure.sql
SELECT [DISTINCT] expression [, col, ... ou *] [[AS] alias]
FROM    table1 [[AS] alias] [, table2, ...]
[WHERE   condition]
[GROUP BY critère]
[HAVING  condition]
[ORDER BY expression [ASC/DESC]]
Les opérateurs de la clause WHERE
OpérateurSignification
=Égale à
<>Différent de
>Supérieur à
>=Supérieur ou égal à
<Inférieur à
<=Inférieur ou égal à
OpérateurSignification
INValeur dans une liste : IN (1, 2, 3)
BETWEENEntre deux valeurs : BETWEEN 10 AND 20
LIKERecherche de motif : LIKE 'A%'
IS NULLValeur nulle
ANDET logique
OROU logique

Exemples SELECT progressifs
SELECT_exemples.sql
-- Toutes les colonnes de tous les élèves
SELECT * FROM eleve;

-- Seulement le nom et la note
SELECT nom, note FROM eleve;

-- Élèves avec note >= 12, triés par note décroissante
SELECT nom, prenom, note
FROM   eleve
WHERE  note >= 12
ORDER BY note DESC;

-- Élèves dont le nom commence par 'A'
SELECT * FROM eleve
WHERE  nom LIKE 'A%';

-- Élèves de la classe 1 OU 2
SELECT * FROM eleve
WHERE  id_classe IN (1, 2);

-- Sans doublons (DISTINCT)
SELECT DISTINCT id_classe FROM eleve;

-- Alias de colonne
SELECT nom AS "Nom de l'élève", note AS "Note /20"
FROM eleve;

Chapitre 5 · Modifier les données

INSERT · UPDATE · DELETE

Ces trois commandes DML permettent d'ajouter, modifier et supprimer des données dans les tables.

INSERT — Insérer des données
INSERT.sql
-- Syntaxe : INSERT INTO table [(colonnes)] VALUES (valeurs)

-- Insertion avec toutes les colonnes
INSERT INTO eleve (nom, prenom, note, id_classe)
VALUES ('Ben Salah', 'Rim', 16.5, 1);

-- Insertion de plusieurs lignes en une seule requête
INSERT INTO eleve (nom, prenom, note, id_classe)
VALUES
  ('Hamdi',   'Ali',   14.0, 2),
  ('Mansour', 'Nour',  17.5, 1),
  ('Zaied',   'Taha',  11.0, 2);
UPDATE — Modifier des données
⚠️ Attention : Sans clause WHERE, UPDATE modifie toutes les lignes de la table. Toujours préciser la condition !
UPDATE.sql
-- Syntaxe : UPDATE table SET colonne = valeur [WHERE condition]

-- Modifier la note de l'élève avec id_eleve = 3
UPDATE eleve
SET    note = 13.5
WHERE  id_eleve = 3;

-- Augmenter de 1 point toutes les notes < 10
UPDATE eleve
SET    note = note + 1
WHERE  note < 10;

-- Modifier plusieurs colonnes en même temps
UPDATE eleve
SET    nom = 'Trabelsi', id_classe = 2
WHERE  id_eleve = 5;
DELETE — Supprimer des données
⚠️ Attention : Sans WHERE, DELETE supprime tous les enregistrements de la table (la structure reste). Utilise toujours une condition précise !
DELETE.sql
-- Syntaxe : DELETE FROM table [WHERE condition]

-- Supprimer l'élève avec id_eleve = 5
DELETE FROM eleve
WHERE  id_eleve = 5;

-- Supprimer tous les élèves avec note < 5
DELETE FROM eleve
WHERE  note < 5;

Chapitre 6 · Agrégation

Fonctions d'agrégation & GROUP BY

Les fonctions d'agrégation effectuent des calculs sur des groupes de lignes et retournent une valeur résumée.

Les fonctions d'agrégation
COUNT(*)
Retourne le nombre total d'enregistrements (lignes).
SELECT COUNT(*) FROM eleve;
AVG(col)
Retourne la moyenne des valeurs de la colonne.
SELECT AVG(note) FROM eleve;
MAX(col)
Retourne la valeur maximale de la colonne.
SELECT MAX(note) FROM eleve;
MIN(col)
Retourne la valeur minimale de la colonne.
SELECT MIN(note) FROM eleve;
SUM(col)
Retourne la somme de toutes les valeurs de la colonne.
SELECT SUM(note) FROM eleve;
GROUP BY et HAVING
GROUP BY regroupe les lignes ayant la même valeur dans une colonne. HAVING filtre les groupes (comme WHERE, mais pour les groupes).
GROUP_BY.sql
-- Nombre d'élèves par classe
SELECT   id_classe, COUNT(*) AS "Nb élèves"
FROM     eleve
GROUP BY id_classe;

-- Moyenne des notes par classe
SELECT   id_classe, AVG(note) AS "Moyenne"
FROM     eleve
GROUP BY id_classe;

-- Classes dont la moyenne >= 12 (HAVING filtre les groupes)
SELECT   id_classe, AVG(note) AS "Moyenne"
FROM     eleve
GROUP BY id_classe
HAVING   AVG(note) >= 12;

Chapitre 7 · Relations entre tables

Jointures & Relations

Les jointures permettent de combiner des données de plusieurs tables en une seule requête, grâce aux clés primaires et étrangères.

// Pourquoi des jointures ?

Dans une base relationnelle, les données sont réparties dans plusieurs tables pour éviter la redondance. Les jointures permettent de les relier au moment de la requête. On joint deux tables sur la colonne commune (souvent PK = FK).

La jointure (requête multi-tables)
jointures.sql
-- Jointure simple : afficher nom de l'élève ET libellé de sa classe
SELECT   e.nom, e.prenom, e.note, c.libelle
FROM     eleve AS e, classe AS c
WHERE    e.id_classe = c.id_classe;

-- Avec filtre : élèves de la classe "Bac STI" avec note >= 12
SELECT   e.nom, e.prenom, e.note
FROM     eleve AS e, classe AS c
WHERE    e.id_classe = c.id_classe
AND      c.libelle = 'Bac STI'
AND      e.note >= 12
ORDER BY e.note DESC;

-- Nombre d'élèves par classe (jointure + agrégation)
SELECT   c.libelle, COUNT(e.id_eleve) AS "Effectif"
FROM     eleve AS e, classe AS c
WHERE    e.id_classe = c.id_classe
GROUP BY c.libelle;
