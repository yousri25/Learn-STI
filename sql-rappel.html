③Fonctions d'agrégation

FonctionDescriptionExemple
AVG(col) Retourne la moyenne des valeurs d'une colonne. SELECT AVG(note) FROM eleves;
COUNT(col) Retourne le nombre d'enregistrements. SELECT COUNT(*) FROM eleves;
MAX(col) Retourne le maximum des valeurs d'une colonne. SELECT MAX(note) FROM eleves;
MIN(col) Retourne le minimum des valeurs d'une colonne. SELECT MIN(note) FROM eleves;
SUM(col) Retourne la somme des valeurs d'une colonne. SELECT SUM(montant) FROM commandes;

④Fonctions définies sur le type DATE

FonctionDescriptionExemple
DATE(expr) Retourne une date extraite à partir d'une chaîne contenant une valeur au format DATE ou DATETIME. SELECT DATE(NOW());
DAY(date) Retourne le jour du mois. SELECT DAY('2024-09-15'); → 15
MONTH(date) Retourne le mois. SELECT MONTH('2024-09-15'); → 9
YEAR(date) Retourne l'année. SELECT YEAR('2024-09-15'); → 2024
NOW() Retourne la date et l'heure courantes. SELECT NOW();
ADDDATE(date, interval) Ajoute un nombre de jours ou un intervalle de temps à une date. SELECT ADDDATE('2024-01-01', 30);
DATEDIFF(d1, d2) Retourne la différence entre deux dates en jours. SELECT DATEDIFF('2024-12-31','2024-01-01'); → 365

⑤Contraintes d'intégrité

ContrainteDescriptionExemple
NOT NULL Interdit une valeur nulle — le champ est obligatoire. nom VARCHAR(50) NOT NULL
DEFAULT val Attribue une valeur par défaut si aucune n'est fournie. statut VARCHAR(20) DEFAULT 'actif'
CHECK (condition) Spécifie que les valeurs de la colonne doivent respecter une condition. note DECIMAL CHECK (note >= 0 AND note <= 20)
PRIMARY KEY Définit une clé primaire — valeur unique et non nulle, identifie chaque enregistrement. id INT PRIMARY KEY
UNIQUE Spécifie l'unicité des valeurs d'une colonne (pas de doublons). email VARCHAR(100) UNIQUE
FOREIGN KEY Définit une clé étrangère — référence la clé primaire d'une autre table. FOREIGN KEY (id_classe) REFERENCES classe(id)
REFERENCES table(col) Fait référence à une clé primaire d'une autre table (utilisé avec FOREIGN KEY). REFERENCES eleves(id)
ON UPDATE CASCADE Met à jour automatiquement la clé étrangère en cas de mise à jour de la clé primaire. ON UPDATE CASCADE
ON DELETE CASCADE Supprime automatiquement tous les enregistrements de la table fille suite à la suppression de l'enregistrement correspondant de la table mère. ON DELETE CASCADE

⑥Définition des données (DDL)

DDL = Data Definition Language — permet de créer, modifier et supprimer la structure des bases de données et tables.
InstructionDescription
CREATE DATABASE nom Crée une base de données.
DROP DATABASE nom Supprime une base de données.
CREATE TABLE table1 (
  col1 type [contrainte],
  col2 type [contrainte],
  [CONSTRAINT] [nom] [contrainte]
) 		Crée une table avec ses colonnes et contraintes.
DROP TABLE table1 Supprime une table et toutes ses données.
ALTER TABLE table1
ADD colonne type [contrainte]		 Ajoute une colonne à une table existante.
ALTER TABLE table1
DROP colonne		 Supprime une colonne d'une table.
ALTER TABLE table1
ALTER* colonne type		 Modifie le type d'une colonne.
* on peut utiliser MODIFY au lieu de ALTER
ALTER TABLE table1
RENAME** nom1 TO nom2		 Modifie le nom d'une colonne.
** on peut utiliser CHANGE au lieu de RENAME
ALTER TABLE table1
ADD CONSTRAINT [nom] contrainte		 Ajoute une contrainte sur une colonne d'une table.
ALTER TABLE table1
DROP CONSTRAINT nom		 Supprime une contrainte.
ALTER TABLE table1
ENABLE / DISABLE CONSTRAINT nom		 Active ou désactive une contrainte.
Exemple complet de création de table avec contraintes :
CREATE TABLE eleve ( id INT PRIMARY KEY, nom VARCHAR(50) NOT NULL, prenom VARCHAR(50) NOT NULL, email VARCHAR(100) UNIQUE, note DECIMAL(4,2) DEFAULT 0 CHECK (note BETWEEN 0 AND 20), id_cls INT NOT NULL, CONSTRAINT fk_classe FOREIGN KEY (id_cls) REFERENCES classe(id) ON DELETE CASCADE ON UPDATE CASCADE );

⑦Manipulation des données (DML)

DML = Data Manipulation Language — permet de sélectionner, insérer, modifier et supprimer des données.

🔍 SELECT — Interrogation

ClauseRôle
SELECT [DISTINCT] col1, col2 …Spécification des colonnes du résultat. DISTINCT évite les doublons.
FROM table1 [[AS] alias] [, table2, …]Spécification des tables source.
[WHERE condition]Filtre les lignes selon une condition.
[GROUP BY critère]Organisation du résultat en groupes.
[HAVING condition]Application d'un filtre sur les groupes (après GROUP BY).
[ORDER BY expression [ASC/DESC]]Tri du résultat par ordre croissant (ASC) ou décroissant (DESC).
-- Syntaxe complète SELECT SELECT [DISTINCT] expression [, col, ... ou *] [[AS] alias] FROM table1 [[AS] alias] [, table2, ...] [WHERE condition] [GROUP BY critère] [HAVING condition] [ORDER BY expression [ASC/DESC]];

➕ INSERT — Insertion

InstructionDescription
INSERT INTO table [(liste_col)]
VALUES (liste_valeurs) 		Insertion des données dans une table.
INSERT INTO eleve (id, nom, prenom, note) VALUES (1, 'Ben Ali', 'Mohamed', 17.5);

✏️ UPDATE — Modification

InstructionDescription
UPDATE table1
SET colonne = expression
[WHERE condition] 		Modification des valeurs d'une ou plusieurs colonnes. Sans WHERE, toutes les lignes sont modifiées.
UPDATE eleve SET note = 18 WHERE id = 1;

🗑️ DELETE — Suppression

InstructionDescription
DELETE FROM table1
[WHERE condition] 		Suppression des enregistrements d'une table. Sans WHERE, toutes les lignes sont supprimées.
DELETE FROM eleve WHERE id = 1;

⑧Exemples complets

Utilisation des opérateurs de comparaison et logiques dans WHERE :
-- Opérateurs de comparaison SELECT * FROM eleve WHERE note >= 10; SELECT * FROM eleve WHERE note BETWEEN 10 AND 20; SELECT * FROM eleve WHERE ville IN ('Tunis', 'Sfax', 'Sousse'); SELECT * FROM eleve WHERE nom LIKE 'Ben%'; SELECT * FROM eleve WHERE email IS NULL; -- Opérateurs logiques SELECT * FROM eleve WHERE note >= 10 AND ville = 'Tunis'; SELECT * FROM eleve WHERE note < 10 OR note IS NULL; SELECT * FROM eleve WHERE NOT (ville = 'Tunis');
Utilisation des fonctions d'agrégation avec GROUP BY et HAVING :
-- Moyenne des notes par classe SELECT id_cls, AVG(note) AS moyenne FROM eleve GROUP BY id_cls HAVING AVG(note) >= 12 ORDER BY moyenne DESC; -- Nombre d'élèves par ville SELECT ville, COUNT(*) AS nb_eleves FROM eleve GROUP BY ville; -- Note max, min et somme SELECT MAX(note), MIN(note), SUM(note) FROM eleve;
Utilisation des fonctions de date :
-- Élèves nés en 2005 SELECT * FROM eleve WHERE YEAR(date_naissance) = 2005; -- Élèves nés au mois de septembre SELECT * FROM eleve WHERE MONTH(date_naissance) = 9; -- Différence de jours entre inscription et aujourd'hui SELECT nom, DATEDIFF(NOW(), date_inscription) AS jours FROM eleve; -- Ajouter 30 jours à une date SELECT ADDDATE(date_inscription, 30) AS date_limite FROM eleve;
Jointure entre deux tables (exemple avec FOREIGN KEY) :
-- Afficher les élèves avec le nom de leur classe SELECT e.nom, e.prenom, e.note, c.nom_classe FROM eleve e, classe c WHERE e.id_cls = c.id ORDER BY e.nom ASC;
Exemples DDL — Modification de structure :
-- Ajouter une colonne ALTER TABLE eleve ADD telephone VARCHAR(20); -- Supprimer une colonne ALTER TABLE eleve DROP telephone; -- Modifier le type d'une colonne ALTER TABLE eleve MODIFY nom VARCHAR(100); -- Ajouter une contrainte CHECK ALTER TABLE eleve ADD CONSTRAINT ck_note CHECK (note BETWEEN 0 AND 20); -- Supprimer une contrainte ALTER TABLE eleve DROP CONSTRAINT ck_note; -- Désactiver / Activer une contrainte ALTER TABLE eleve DISABLE CONSTRAINT ck_note; ALTER TABLE eleve ENABLE CONSTRAINT ck_note;
