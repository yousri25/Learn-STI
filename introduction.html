①C'est quoi un site web ?

Un site web est un ensemble de pages accessibles via un navigateur (Chrome, Firefox…) grâce à une adresse URL. Ces pages sont construites avec des langages spécialisés qui ont chacun un rôle précis.
Type de siteDescriptionTechnologies
Site statique Le contenu est fixe — tous les visiteurs voient la même page. Aucun traitement serveur, aucune base de données. HTML5 · CSS3 · JS (optionnel)
Site dynamique Le contenu est généré à la volée selon l'utilisateur, ses actions, ou les données stockées. Chaque visiteur peut voir un contenu différent. HTML5 · CSS3 · JS · PHP · SQL

②Le Front-end — ce que voit l'utilisateur

Le front-end est la partie visible du site : tout ce qui s'affiche dans le navigateur de l'utilisateur. Il est construit avec trois technologies complémentaires qui travaillent ensemble.
HTML5
Définit la structure et le contenu de la page : titres, paragraphes, images, formulaires, liens, tableaux.
Navigateur · Front-end
CSS3
Gère l'apparence visuelle : couleurs, polices, marges, animations, mise en page responsive.
Navigateur · Front-end
JavaScript
Ajoute l'interactivité : validation de formulaires, modification du DOM, réactions aux événements utilisateur.
Navigateur · Front-end
Comment ça fonctionne ? Le navigateur envoie une requête HTTP au serveur, qui répond avec des fichiers HTML/CSS/JS. Le navigateur les interprète et affiche la page. Pour un site statique, c'est tout — le serveur envoie simplement les fichiers tels quels.

Mais comment on sauvegarde les données ?

Un formulaire d'inscription, des produits, des messages… où vont ces informations ?

③Les données — Base de données & SQL

Pour stocker des données de manière persistante (utilisateurs, produits, commandes…), on utilise une base de données. Le langage utilisé pour la manipuler s'appelle SQL.
SQL — Structured Query Language
Langage pour créer des tables, insérer, modifier, supprimer et interroger des données. Les commandes clés : SELECT, INSERT, UPDATE, DELETE.
Serveur · Base de données MySQL

Pour développer en local (sur ton ordinateur), on utilise XAMPP — un package qui installe en une seule fois tout ce dont on a besoin : un serveur web, PHP et MySQL.

Comment relier le front-end à la base de données ?

HTML/CSS/JS s'exécutent dans le navigateur. MySQL s'exécute sur le serveur. Comment les faire communiquer ?

④Le Back-end — PHP comme pont

C'est le rôle de PHP. PHP est un langage exécuté côté serveur (pas dans le navigateur). Il reçoit les données envoyées par le formulaire HTML, les traite, interroge la base de données MySQL, et génère dynamiquement la page HTML à renvoyer.
PHP — Hypertext Preprocessor
Reçoit les données via $_POST / $_GET. Applique la logique métier. Interroge MySQL via MySQLi. Génère le HTML dynamique retourné au navigateur.
Serveur Apache · Back-end
PHP fait quoi ?Explication
Reçoit les donnéesVia $_POST['nom'] ou $_GET['id'] après soumission d'un formulaire HTML.
Traite la logiqueVérifie les données, applique les règles métier (conditions, boucles, calculs).
Parle à MySQLUtilise mysqli_connect(), mysqli_query() pour lire/écrire dans la base de données.
Génère le HTMLProduit une page HTML complète avec les données récupérées, envoyée au navigateur.
Gère les sessionsMaintient l'état de connexion d'un utilisateur entre les pages via $_SESSION.
