Apprendre HTML5

Un guide pas à pas pour comprendre et maîtriser les bases du web

Chapitre 1 · Les bases

C'est quoi HTML ?

Avant d'écrire la moindre ligne de code, comprenons ce qu'est vraiment HTML et pourquoi c'est le point de départ de tout site web.

// Définition

HTML signifie HyperText Markup Language. C'est un langage de balisage — pas un langage de programmation. Son rôle est de décrire la structure et le contenu d'une page web à l'aide de balises.

💡 Analogie

Imagine que tu construis une maison. HTML c'est le gros œuvre : les murs, les portes, les fenêtres. Il dit ce qui existe et où ça se trouve, mais pas encore à quoi ça ressemble. Ce sera le rôle du CSS plus tard.

À quoi sert HTML ?
📄
Structurer le texte
Titres, paragraphes, listes…
🖼️
Intégrer des médias
Images, vidéos, audio…
🔗
Créer des liens
Naviguer entre les pages
📝
Créer des formulaires
Saisir et envoyer des données
📐
Organiser la mise en page
Sections, colonnes, blocs…
♿
Accessibilité
Structure lisible par tous
Comment ça fonctionne ?
Un fichier HTML est un simple fichier texte avec l'extension .html. Le navigateur (Chrome, Firefox…) le lit et l'interprète pour afficher la page. Tu n'as pas besoin d'installer quoi que ce soit pour commencer — juste un éditeur de texte et un navigateur.
Le principe des balises
HTML fonctionne avec des balises (tags en anglais). Une balise est un mot-clé entouré de chevrons < >. La plupart des balises vont par paires : une balise ouvrante et une balise fermante.
Anatomie d'une balise HTML
<p>Bonjour le monde !</p>
↑ Balise ouvrante — commence l'élément
               ↑ Contenu — ce qui sera affiché
                            ↑ Balise fermante — ferme avec </
Balise avec attribut :
<img src="photo.jpg" alt="Ma photo">
→
Ce que voit l'utilisateur

Bonjour le monde !


↑ Le navigateur lit la balise <p> et affiche le texte comme un paragraphe.

Chapitre 2 · Le squelette

La structure de base

Chaque page HTML doit respecter une structure précise. Voyons ce squelette obligatoire et ce que fait chaque partie.

Tout fichier HTML valide commence par la même structure. Comme un être humain a toujours un squelette, une page web a toujours ce squelette HTML.
Le squelette complet
index.html
<!DOCTYPE html>
<html lang="fr">

  <head>
    <meta charset="UTF-8">
    <title>Ma première page</title>
  </head>

  <body>
    <h1>Bonjour !</h1>
    <p>C'est ma première page web.</p>
  </body>

</html>
Que fait chaque ligne ?
<!DOCTYPE html>
Déclare au navigateur que ce fichier est un document HTML5. C'est toujours la toute première ligne — obligatoire, mais ce n'est pas vraiment une balise HTML.
<html> racine
La balise racine qui contient tout le document. L'attribut lang="fr" indique la langue — important pour l'accessibilité.
<head> invisible
Contient les informations sur la page : titre, encodage, liens CSS, scripts. Son contenu n'est pas visible dans le navigateur.
<meta charset="UTF-8">
<title>Mon titre</title>
<meta charset>
Définit l'encodage des caractères. UTF-8 permet d'afficher correctement les accents français (é, è, à, ç…). À toujours inclure.
<title>
Définit le titre de l'onglet du navigateur. C'est aussi ce que les moteurs de recherche affichent dans les résultats.
<body> visible
Contient tout le contenu visible de la page : textes, images, liens, formulaires. C'est là que tout se passe !
L'imbrication des balises
💡 Règle importante

Les balises HTML s'imbriquent comme des poupées russes. Une balise ouverte à l'intérieur d'une autre doit être fermée avant de fermer la balise parente. Toujours respecter l'ordre de fermeture.

Imbrication correcte vs incorrecte
✅ Correct
<p>
  <strong>Texte</strong>
</p>
vs
❌ Incorrect
<p>
  <strong>Texte</p>
</strong> ← Erreur !

Chapitre 3 · Le contenu

Les balises essentielles

Maintenant que tu connais la structure, découvrons les balises les plus utilisées pour créer du contenu dans le corps de la page.

Les titres — <h1> à <h6>
HTML propose 6 niveaux de titres. <h1> est le plus important (une seule fois par page), <h6> est le moins important. Ils créent une hiérarchie de contenu.
titres.html
<h1>Titre principal (le plus grand)</h1>
<h2>Sous-titre</h2>
<h3>Sous-sous-titre</h3>
<h4>Niveau 4</h4>
Paragraphes et formatage du texte
<p>
Définit un paragraphe. Le navigateur ajoute automatiquement un espace avant et après.
<p>Mon paragraphe.</p>
<br> vide
Retour à la ligne sans créer un nouveau paragraphe. Balise vide — pas de balise fermante.
Ligne 1<br>Ligne 2
<strong>
Texte en gras. Indique une importance forte pour le contenu.
<strong>Important</strong>
<em>
Texte en italique. Indique une emphase sur le mot.
<em>Accentué</em>
<mark>
Texte surligné en jaune. Utile pour mettre en évidence un mot clé.
<mark>Mot clé</mark>
<hr> vide
Ligne horizontale de séparation. Représente un changement thématique dans le contenu.
<hr>
Les listes
HTML propose deux types de listes : non ordonnée (à puces) avec <ul>, et ordonnée (numérotée) avec <ol>. Chaque élément est une balise <li>.

Les images — <img>
La balise <img> est une balise vide (pas de balise fermante). Elle a deux attributs indispensables : src (chemin de l'image) et alt (description alternative pour l'accessibilité).
image.html
<!-- Image locale -->
<img src="photo.jpg" alt="Description de la photo" width="300">

<!-- Image avec figure et légende -->
<figure>
  <img src="photo.jpg" alt="Mon image">
  <figcaption>Voici la légende de l'image</figcaption>
</figure>
Les tableaux
Un tableau HTML se construit avec plusieurs balises imbriquées. La hiérarchie est : <table> → <thead> / <tbody> → <tr> (ligne) → <th> (en-tête) / <td> (cellule).

Chapitre 4 · Interaction

Formulaires & Liens

Les formulaires et les liens sont les deux piliers de l'interaction dans une page web. Ils permettent de naviguer et de collecter des données.

Les liens hypertexte — <a>
La balise <a> crée un lien. L'attribut href contient la destination. L'attribut target="_blank" ouvre le lien dans un nouvel onglet.
Lien externe
Vers un autre site web. Commence par https://
<a href="https://google.com">Google</a>
Lien interne
Vers une autre page du même site.
<a href="contact.html">Contact</a>
Lien ancre
Vers une section dans la même page, via un id.
<a href="#section1">Aller à...</a>
Nouvel onglet
Ouvre le lien dans un nouvel onglet.
<a href="..." target="_blank">Ouvrir</a>
Les formulaires — <form>
Un formulaire collecte des données et les envoie au serveur. Les attributs action (où envoyer) et method (get ou post) sont essentiels.
formulaire.html
<form action="traitement.php" method="post">

  <!-- Champ texte -->
  <label for="nom">Nom :</label>
  <input type="text" id="nom" name="nom" placeholder="Votre nom" required>

  <!-- Email -->
  <label for="email">Email :</label>
  <input type="email" id="email" name="email" required>

  <!-- Liste déroulante -->
  <select name="classe">
    <option value="4sti">4ème STI</option>
    <option value="bac">Bac STI</option>
  </select>

  <!-- Bouton d'envoi -->
  <input type="submit" value="Envoyer">

</form>
Les types de champs <input>
La balise <input> change de comportement selon la valeur de l'attribut type. C'est la balise la plus polyvalente des formulaires.
type="text"
Champ de saisie de texte libre.
<input type="text" placeholder="Nom">
type="email"
Champ email — valide le format automatiquement.
<input type="email">
type="password"
Masque les caractères saisis.
<input type="password">
type="number"
Champ numérique avec min et max.
<input type="number" min="0" max="20">
type="checkbox"
Case à cocher — plusieurs choix possibles.
<input type="checkbox" checked>
type="radio"
Bouton radio — un seul choix parmi plusieurs.
<input type="radio" name="choix">
type="date"
Sélecteur de date avec calendrier intégré.
<input type="date">
type="submit"
Bouton d'envoi du formulaire.
<input type="submit" value="Envoyer">

Chapitre 5 · L'écosystème web

HTML + CSS + JavaScript

HTML ne travaille jamais seul. Découvre comment les trois technologies du front-end collaborent pour créer une page web complète.

// Le trio du front-end

Chaque technologie a un rôle précis et complémentaire. Elles s'appliquent dans le navigateur, dans cet ordre : HTML en premier, CSS ensuite, JavaScript par-dessus.

</>
HTML5
La structure
Définit ce qui existe sur la page — les éléments et leur contenu. C'est le squelette.
<button id="btn">Cliquer</button>
CSS
CSS3
L'apparence
Définit comment ça ressemble — couleurs, polices, tailles, animations.
#btn { background: green;
  color: white; }
JS
JavaScript
Le comportement
Définit ce qui se passe — réactions aux clics, animations, validation.
btn.onclick = function() {
  alert("Cliqué !");
}
💡 Analogie — la maison

HTML = les murs, les portes, les fenêtres (la structure).
CSS = la peinture, les meubles, la décoration (l'apparence).
JavaScript = l'électricité, la domotique — tout ce qui bouge et réagit (le comportement).

Voir les trois ensemble
Voici comment HTML, CSS et JS se combinent dans un seul fichier. En pratique, CSS et JS sont souvent dans des fichiers séparés (style.css et script.js).
exemple-complet.html
<!DOCTYPE html>
<html lang="fr">
<head>
  <!-- CSS dans le <head> -->
  <style>
    h1 { color: #00a87e; }
    button { background: #00c896; color: white;
             padding: 10px 20px; border: none; cursor: pointer; }
  </style>
</head>
<body>

  <!-- HTML — la structure -->
  <h1>Compteur</h1>
  <p id="compte">0</p>
  <button onclick="incrementer()">+ 1</button>

  <!-- JavaScript en bas du <body> -->
  <script>
    let n = 0;
    function incrementer() {
      n++;
      document.getElementById('compte').innerHTML = n;
    }
  </script>

</body>
</html>

Structurer sa page avec les balises sémantiques
HTML5 a introduit des balises sémantiques pour mieux organiser la page. Elles donnent du sens à la structure, améliorent le référencement et l'accessibilité.
<header>
En-tête de la page ou d'une section. Contient souvent le logo et la navigation.
<nav>
Zone de navigation — menus, liens internes.
<main>
Contenu principal — un seul par page.
<section>
Section thématique avec un titre.
<article>
Contenu autonome et indépendant.
<footer>
Pied de page — copyright, liens légaux.
